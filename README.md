# Aviation Service API

## Тема

**Авиаперевозки**: рейсы, самолёты, аэропорты, пассажиры и бронирование. Рейс использует самолёт и аэропорты отправления/прибытия; бронирование закрепляет место за пассажиром. Количество мест не превышает вместимость самолёта; статус рейса влияет на возможность брони и возврата.

## Основные сущности

- **Aircraft** — самолёт (модель, бортовой номер, вместимость)
- **Airport** — аэропорт (код, название, город)
- **Flight** — рейс (самолёт, аэропорт вылета, аэропорт прилёта, время, статус)
- **Passenger** — пассажир (ФИО, email, номер паспорта)
- **Booking** — бронирование (пассажир, рейс, номер места)

## Операции сервиса

### CRUD по сущностям

- **Aircraft**: создание, получение по id, список, обновление, удаление
- **Airport**: создание, получение по id, список, обновление, удаление
- **Flight**: создание, получение по id, список, обновление, удаление
- **Passenger**: создание, получение по id, список, обновление, удаление
- **Booking**: создание, получение по id, список (с фильтром по flightId/passengerId), обновление, удаление

### Бизнес-операции (не только CRUD)

1. **Оформить бронирование** — `POST /api/bookings` (проверка рейса, пассажира, свободных мест, статуса рейса; одна транзакция)
2. **Отменить бронирование (возврат)** — `DELETE /api/bookings/{id}` (проверка статуса рейса; транзакция)
3. **Рейсы по маршруту** — `GET /api/operations/flights-by-route?departure=SVO&arrival=LED` (рейсы по кодам аэропортов вылета и прилёта)
4. **Свободные места по рейсу** — `GET /api/operations/flights/{id}/available-seats` (вместимость, число забронированных мест, список занятых мест)
5. **Отмена рейса** — `POST /api/operations/flights/{id}/cancel` (отмена рейса и всех бронирований в одной транзакции)



Чувствительные данные (URL БД, пользователь, пароль) задаются через переменные окружения; пример — в файле `env.example`. В репозиторий не будут коммититься файлы с реальными паролями.
